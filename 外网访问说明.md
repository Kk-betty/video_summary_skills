# 视频总结网站 - 外网访问指南

## 🎉 当前状态

✅ 服务已配置为支持外网访问
✅ 前端和后端已启动

---

## 🌐 方案一：局域网访问（同一WiFi下）

如果您和朋友在同一个WiFi网络下，可以直接使用局域网地址访问：

**访问地址：**
```
http://192.168.1.5:5173
```

（实际地址请查看前端启动输出中的 Network 地址）

---

## 🚀 方案二：使用内网穿透工具（推荐，可在任何网络访问）

### 选项 A：使用 localtunnel（最简单，无需注册）

1. 新开一个终端，进入项目目录
2. 运行以下命令：

```bash
node start-localtunnel.js
```

或者直接使用 npx：

```bash
npx localtunnel --port 5173
```

3. 等待几秒钟，您会获得一个外网访问链接，例如：
   ```
   your url is: https://fancy-panda-123.loca.lt
   ```

4. 将此链接分享给朋友即可！

---

### 选项 B：使用 ngrok（稳定，需要简单注册）

1. 新开一个终端，进入项目目录
2. 运行以下命令：

```bash
node start-ngrok.js
```

或者直接使用 npx：

```bash
npx ngrok http 5173
```

3. 等待几秒钟，您会获得一个外网访问链接

---

## 📦 一键启动所有服务

如果需要重新启动服务，可以使用：

```bash
node start-all.js
```

这会同时启动：
- ✅ 后端 API 服务器（端口 3001）
- ✅ 前端 Vite 服务器（端口 5173）

---

## 🔧 配置说明

### 已修改的文件：

1. **vite.config.ts** - 前端配置
   - 添加 `host: '0.0.0.0'` - 监听所有网络接口
   - 添加 `port: 5173` - 固定端口

2. **api/server.js** - 后端配置
   - 修改为监听 `'0.0.0.0'` - 支持外网访问

---

## ⚠️ 注意事项

1. **安全性**：外网访问时请注意不要分享敏感信息
2. **稳定性**：内网穿透工具免费版可能有连接数限制
3. **IP变更**：每次重新启动内网穿透工具，链接可能会变化
4. **防火墙**：确保您的防火墙允许相关端口的连接

---

## 💡 推荐方案

**如果是临时分享给朋友：**
→ 使用 `localtunnel`（最简单快捷）

**如果需要长期稳定访问：**
→ 使用 `ngrok`（更稳定）

**如果有自己的云服务器：**
→ 使用 `frp` 或直接部署到云服务器

---

## 🆘 常见问题

**Q: localtunnel 连接失败怎么办？**
A: 尝试使用 ngrok 作为替代方案

**Q: 链接访问很慢怎么办？**
A: 这是正常的，因为数据需要经过中继服务器

**Q: 可以使用自定义域名吗？**
A: 付费版的 ngrok 或 localtunnel 支持自定义域名
